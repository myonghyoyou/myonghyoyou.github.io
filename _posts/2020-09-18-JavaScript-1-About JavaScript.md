### 정의

자바스크립트는 '웹페이지에 생동감을 불어넣기 위해' 만들어진 프로그래밍 언어이다. HTML과 CSS만으로 구성된 웹페이지는 정적인 페이지이기 때문에,
거기에 동적인 움직임을 넣어주기 위해 자바스크립트를 사용한다.

자바스크립트로 작성한 프로그램을 스크립트(script) 라고 부른다. 스크립트는 웹페이지의 HTML 안에 작성할 수 있는데, 웹페이지를 불러올 때 자동으로 실행된다.

스크립트는 특별한 준비나 컴파일 없이 보통의 문자 형태로 작성할 수 있으며, 실행도 가능하다.

이런 관점에서 보면 자바스크립트는 자바(Java)와는 매우 다른 언어이다.

>왜 ___자바___ 스크립트일까?
>>처음에는 LiveScript로 불리던 자바스크립트는 당시 인기가 아주 많던 자바의 '동생'격인 언어로 홍보되면 도움이 될 것 같다는 관련인들의 판단 하에 '_자바스크립트_' 라는 이름을 가지게 되었다.
이름은 자바에서 차용해왔으나, 자바와는 전혀 상관이 없으며 지금은 ECMAScript라는 고유한 이름을 갖춘 독립적인 언어로 자리 잡았다.

자바스크립트는 브라우저뿐만 아니라 서버에서도 실행 가능하다. 그 외에도 자바스크립트 엔진이라 불리는 특별한 프로그램이 있는 모든 디바이스에서 동작한다.

브라우저에는 '자바스크립트 가상 머신' 이라 불리는 엔진이 내장되어 있다.

엔진의 종류는 다양하며, 엔진마다 특유의 코드네임이 있다. 아래를 보자.

* V8 : Chrome과 Opera에서 쓰임
* SpiderMonkey : Firefox에서 사용
* IE는 버전에 따라 'Trident' / 'Chakra' 라 불리는 엔진 사용. 'ChakraCor'는 Edge에서 사용되며, 'SquirrelFish'는  Safari에서 사용된다.

### 자바스크립트가 브라우저에서 할 수 있는 일

모던 자바스크립트는 안전한 프로그래밍 언어이다. 메모리나 CPU 같은 저수준 영역의 조작을 허용하지 않는다. 애초에 이러한 접근이 필요하지 않은 브라우저를 대상으로 만든 언어이기 때문이다.

JS의 능력은 실행 환경에 상당한 영향을 받는다. 아래 예시를 통해 살펴보자.

* Node.js 환경 : 임의의 파일을 읽거나 쓰고, 네트워크 요청을 수행하는 함수를 지원
* 브라우저 환경 : 웹페이지 조작, 클라이언트와 서버 간 상호 작용에 관한 모든 작업

다음은 브라우저에서 자바스크립트로 할 수 있는 일들이다.

* 페이지에 새로운 HTML 추가 / 기존의 HTML 혹은 스타일 수정
* 마우스 클릭이나 포인터의 움직임, 키보드 키 눌림 등과 같은 사용자 동작에 이벤트 발생
* 네트워크를 통해 원격 서버에 요청을 보내거나, 파일 다운로드 및 업로드 (AJAX나 COMET 같은 기술 이용)
* 쿠키 송신 및 설정. 사용자에게 질문 혹은 메시지 송출.
* 클라이언트 측(로컬 스토리지)에 데이터 저장하기

### 자바스크립트가 브라우저에서 할 수 없는 일

브라우저는 보안을 위해 자바스크립트의 기능에 제약을 걸어놓았다. 이런 제약은 악성 웹페이지가 개인 정보에 접근하거나 사용자의 데이터를 손상하는 것을 방지한다.

아래는 자바스크립트가 브라우저에서 할 수 없는 일들이다.

* 웹페이지 내 스크립트는 디스크에 저장된 임의의 파일을 읽거나 쓰고, 복사하거나 실행할 때 제약을 받을 수 있다. 모던 브라우저를 사용하면 파일을 다룰 순 있으나 접근이 제한된다.
카메라나 마이크와 같은 디바이스와 상호 작용함에 있어서도 사용자의 명시적인 허가를 필요로 한다.
* 브라우저 내 탭과 창은 대개 서로에게서 정보를 캐낼 수 없다. 그러나 JS를 사용해 한 창에서 다른 창을 열 때는 예외가 적용된다. 하지만 이 경우에도 도메인, 프로토콜, 혹은 포트가 다르면
페이지에 접근할 수 없다. 이런 제약사항을 '동일 출처 정책 (Same Origin Policy)'라 부른다. 이 정책을 피하려면 각 페이지는 교환에 동의해야 하며, 동의와 관련된 JS 코드를 포함하고 있어야 한다.
* 자바스크립트를 이용하여 페이지를 생성한 서버와 쉽게 정보를 주고받을 수 있다. 그러나 위에서 언급했듯이 타 사이트나 도메인에서 데이터를 받아오는 것은 불가능하다.
가능하다고 해도 원격 서버에서 HTTP 메시지 등을 통해 허가를 해야 한다.

### 자바스크립트만의 강점

* HTML/CSS와 완전히 통합 가능
* 간단한 일은 간단하게 처리할 수 있음
* 모든 주요 브라우저에서 지원하고, 기본 언어로 사용됨

이 3가지 모두를 지원하는 브라우저 연관 기술은 자바스크립트뿐이다. 이런 특징 때문에 자바스크립트는 브라우저 인터페이스를 만들 때 가장 널리 사용되며 이 외에도 서버나 모바일 앱을 만드는 것도
가능하다.

### 요약

* 자바스크립트는 브라우저에서만 쓸 목적으로 고안된 언어디지만, 지금은 다양한 환경에서 쓰이고 있다.
* 오늘날 자바스크립트는 브라우저 환경에서 가장 널리 사용되는 언어로 자리매김하였다.
* 자바스크립트로 '트랜스파일' 할 수 있는 언어는 많다. 자바스크립트에 숙달한 후에 학습하도록 하자.
