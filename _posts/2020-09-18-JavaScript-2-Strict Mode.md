### 개요

자바스크립트는 오랜 시간 호환성 이슈 없이 발전해왔다. 기존의 기능은 그대로 두면서 새로운 기능이 추가되어온 것이다.

덕분에 기존에 작성한 코드는 절대 망가지지 않는다는 장점이 있으나, 반면에 만들어질 당시의 실수나 결점들이 언어 안에 영원히 존재한다는 단점도 동반한다.

이런 상황은 ECMAScript5(ES5)가 등장하기 전까지 계속 되었다. 그런데 ES5가 나오며 새로운 기능이 추가되고 기본 기능 중 일부가 변경되었다. 기존 기능의 변경으로 인해 하위 호환성 문제가
발생할 것을 우려하여 변경사항 대부분은 ES5의 기본 모드에선 활성화되지 않도록 설계되었다. 대신 `use strict` 라는 특별한 지시자를 사용해 엄격 모드(Strict Mode)를 활성화 했을 때만
이 변경사항이 활성화되도록 했다.

---

### use strict

지시자 `"use strict"` 혹은 `'use strict'`는 단순한 문자열처럼 생겼지만 스크립트 최상단에 오면 스크립트 전체가 "모던 자바스크립트"의 방식으로 작동한다.

``` javascript
"use strict";

// 이 코드는 이제 모던한 방식으로 실행된다.
...
```
 
 "use strict"는 스크립트 최상단이 아닌 함수 본문 맨 앞에 올 수도 있다. 그렇게 하면 오직 해당 함수만 엄격 모드로 실행된다. 그러나 대개 엄격모드는 스크립트 전체에 적용한다.
 >"use script"는 반드시 최상단에 위치해야한다.
 >>`"use script"`가 최상단에 위치하지 않으면 엄격모드가 활성화 되자 않는다. "use script"의 위에는 주석만 사용 가능하다.
 
 <br/>
 
 >`"use script"`를 취소할 방법은 없다.
 >>"no use strict"와 같은 지시자는 존재하지 않기에 일단 엄격 모드가 적용되면 돌이킬 방법은 없다.

### 브라우저 콘솔

브라우저 콘솔을 사용하는 경우에는, 기본적으로 `use strict`가 적용되어 있지 않다.

`use strict`에 영향을 받게 되면 개발자가 예상한 결과를 얻지 못할 수도 있기 때문이다.

그렇기에 엄격 모드(Strict Mode)에 들어가기 위해서는 콘솔에 필요한 텍스트를 입력해줘야 한다.

'use strict'를 입력한 후, `Shift + Enter` 키를 눌러 줄 바꿈한 뒤 원하는 스크립트를 입력하면 엄격 모드(Strict Mode)로 동작하게 된다.

```
'use strict'; <Shift+Enter를 눌러 줄 바꿈 함>

// ... 테스트하려는 코드 입력

<Enter를 눌러 실행>
```

이 기능은 Firefox와 Chrome과 같은 유명 브라우저에서 대부분 실행 가능하다.


---

### 'use strict'는 필수인가?

모던 자바스크립트는 '클래스' 와 '모듈' 이라 불리는 발전한 구조를 제공한다. 이 둘을 사용하면 `use strict`가 자동으로 적용되기에 스크립트에 `"use strict"`를 붙일 필요가 없다.

결론적으로, 코드를 클래스와 모듈을 사용해 구성한다면 `"use strict"`를 생략해도 된다.
