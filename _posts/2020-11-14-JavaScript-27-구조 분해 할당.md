### 개요

객체 혹은 배열을 함수에 전달하고자 할 때, 객체나 배열에 저장된 데이터 전체가 아니라 일부만 필요한 경우가 있을 수 있다.

이럴 때 객체나 배열을 변수로 '분해'할 수 있게 해주는 문법인 *구조 분해 할당(destructing assignment)*을 사용할 수 있다.

---

### 배열 분해하기

배열은 변수로 분해할 수 있다.

예시:

```` javascript
// 이름과 성을 요소로 가진 배열
let arr = ["Bora", "Lee"];

// 구조 분해 할당을 이용
// firstName엔 arr[0], lastName엔 arr[1]을 할당
let [firstName, lastName] = arr;

alert(firstName) // "Bora"
alert(lastName) // "Lee"
````

이렇게 하면 인덱스를 이용해 배열에 접근하지 않고도 변수로 이름과 성을 쓸 수 있게 된다.

아래 예시처럼 `split` 을 이용해 반환 값을 배열로 만들어서 활용할 수도 있다.

``` javascript
let [firstName, lastName] = "Bora Lee".split(" ");
```

##### 분해는 파괴를 의미하지 않는다.

배열을 분해한다는 것은 파괴를 의미하는 것이 아니다. 요소를 복사한 다음에 변수로 분해해준다는 것을 의미하는 것이다. 이 과정에서 분해 대상은 수정되거나 삭제되지 않는다.

배열의 요소를 직접 변수에 할당하는 것보다 코드 양이 줄어든다는 점만 다르다.

##### 쉼표를 사용하여 요소 무시하기

쉼표를 사용하여 필요하지 않은 배열 요소를 버릴 수 있다.

``` javascript
// 두 번째 요소는 필요하지 않다.
let [firstName, , title] = ["Julius", "Caesar", "Consul", "Of the Roman Republic"];

alert(title); // Consul, Caesar가 무시된다.
```

##### 할당 연산자 우측엔 모든 이터러블이 올 수 있다.

배열뿐만 아니라 모든 이터러블에 구조 분해 할당을 적용할 수 있다.

``` javascript
let [a, b, c] = "abc"; // ["a", "b", "c"]
let [one, two, three] = new Set([1,2,3]);
```

##### 할당 연산자 좌측엔 뭐든지 올 수 있다.

할당 연산자 좌측엔 '할당할 수 있는' 것이라면 무엇이든지 올 수 있다.

객체 프로퍼티도 가능하다.

``` javascript
let user = {};
[user.name, user.surname] = "Bora Lee".split(" ");

alert(user.name) // Bora
```

##### .entries()로 반복하기

Objec.entries(obj)와 구조 분해를 조합하면 객체의 키와 값을 순회해 변수로 분해 할당할 수 있다.

``` javascript
let user {
    name: "John",
    age: 30
}

// 객체의 키와 값 순회하기
for (let [key,value] of Object.entries(user)) {
    alert(`${key}:${value}`); // name:John, age:30
}
```

맵에도 적용이 가능하다.

``` javascript
let user = new Map();
user.set("name", "John");
user.set("age", "30");

for (let [key,value] of user) {
    alert(`${key}:${value}`); // name:John, age:30
}
```

##### 변수 교환 트릭

두 변수에 저장된 값을 교환하는 것도 가능하다.

``` javascript
let guest = "Jane";
let admin = "Pete";

// 변수 guest엔 "Pete"가 admin에는 "Jane"이 저장되도록 한다.
[guest, admin] = [admin, guest];

alert(`${guest}, ${admin}`); // Pete, Jane 변수가 교환된 것을 확인할 수 있다.
```

이 방식을 이용하면 두 개뿐만 아니라 그 이상의 변수에 담긴 값도 서로 교환할 수 있다.

### '...'로 나머지 요소 가져오기

배열 앞쪽에 있는 값 몇 개만 필요하고 이어지는 나머지 값들은 한데 모아서 저장하는 것도 가능하다.

`...`를 붙인 매개변수 하나를 추가하면 나머지 요소를 불러올 수 있다.

``` javascript
let [name1, name2, ...rest] = ["Julius", "Caesar", "Consul", "Of the Roman Republic"];

alert(name1); // Julius
alert(name2); // Caesar

// rest는 앞의 두 개 배열을 제외한 나머지 요소들이 저장된 배열이다.
alert(rest[0]); // Consul
alert(rest[1]); // Of the Roman Republic
alert(rest.length); // 2
```

`rest` 대신에 다른 이름을 사용하는 것도 가능하지만, 변수 이름 앞에 `...`을 붙이는 것은 꼭 지켜야 한다.

### 기본값

할당하고자 하는 변수의 개수가 배열의 길이보더 길더라도 오류가 발생하지 않는다. 할당할 값이 없으면 undefined로 취급된다.

``` javascript
let [firstName, lastName] = [];

alert(firstName); // undefined
alert(lastName); // undefined
```

`=`를 이용하면 할당할 값이 없을 때 기본으로 할당해줄 값인 '기본값(default)'를 설정할 수 있다.

``` javascript
// 기본값
let [name = "Guest", surname = "Anonymous"] = ["Julius"];

alert(name); // Julius (배열에서 받아온 값)
alert(surname); // Anonymous (기본값)
```

복잡한 표현식이나 함수 호출도 기본값이 될 수 있다. 기본값으로 표현식이나 함수를 설정하게 되면 할당할 값이 없을 때 표현식이 평가되거나 함수가 호출될 수 있다.

아래 예시는 기본값으로 두 개의 `prompt` 함수를 할당한 모습이다.

``` javascript
// name에 prompt가 실행됨
let [surname = prompt('성을 입력하세요'), name = prompt('이름을 입력하세요')] = ["김"];

alert(surname); // 김
alert(name); // prompt를 통해서 받아온 값
```

### 객체 분해하기

구조 분해 할당으로 객체 또한 분해할 수 있다.

``` javascript
let {var1, var2} = {var1:..., var2:...};
```

할당 연산자 우측엔 분해하고자 하는 객체를, 좌측엔 상응하는 객체 프로퍼티의 '패턴'을 넣는다.

예시:

``` javascript
let options = {
  title: "Menu",
  width: 100,
  height: 200
};

let {title, width, height} = options;

alert(title); // Menu
alert(width); // 100
alert(height); // 200
```

프로퍼티 `options.title`, `options.width`, `options.height`에 저장된 값이 상응하는 변수에 할당된 것을 볼 수 있다. 순서가 바뀌더라도 위 예시와 동일하게 동작하는 것을 볼 수 있다.

``` javascript
// let {...} 안의 순서가 바뀌어도 동일하게 작동한다.
let {height, title, width} = {title: "Menu", height: 200, width: 100};
```

할당 연산자 좌측엔 더 복잡한 패턴이 올 수 있다. 분해하려는 객체의 프로퍼티와 변수의 연결을 원하는대로 조정할 수 있다.

예시:

``` javascript
let options = {
    title : "Menu",
    width : 100,
    height : 200
};

// {객체 프로퍼티: 목표 변수}
let {width: w, height: h, title} = options;

// width -> w
// height -> h
// title -> title

alert(w); // 100
alert(h); // 200
alert(title); // Menu
```

프로퍼티가 없는 경우를 대비하여 `=` 을 사용해 기본값을 설정하는 것도 가능하다.

``` javascript
let options = {
    title: "Menu"
};

let {width = 100, height = 200, title} = options;

alert(title); // Menu
alert(width); // 100
alert(height); // 200
```

객체에도 표현식이나 함수 호출을 기본값으로 할당할 수 있다. 

``` javascript
let options = {
    title: "Menu"
};

let {width = prompt("width?"), title = prompt("title?")} = options;

alert(title); // Menu
alert(width); // prompt에 입력한 값
```

콜론과 할당 연산자를 함께 사용하는 것도 가능하다.

``` javascript
let options = {
    title: "Menu"
}

let {width : w = 100, height : h = 200, title} = options;

alert(title); // Menu
alert(w); // 100
alert(h); // 200
```

프로퍼티가 많은 객체에서 원하는 정보만 뽑아올 수 있다.

``` javascript
let options = {
    title: "Menu",
    width: 100,
    height: 200
}

// title만 변수로 뽑는다.
let {title} = options;

alert(title); // Menu
```

### 나머지 패턴 '...'

배열의 나머지 패턴에서 했던 것과 같이 객체의 나머지 프로퍼티를 어딘가에 할당하는 것이 가능하다.

> IE를 비롯한 구식 브라우저에서는 호환되지 않을 수 있다.

``` javascript
let options = {
    title: "Menu",
    height: 200,
    width: 100
}

// title = 이름이 title인 프로퍼티
// rest = 나머지 프로퍼티
let {title, ...rest} = options;

// rest에 {height: 200, width:100}이 할당된다.
alert(rest.height); // 200
alert(rest.width); // 100
```

### 중첩 구조 분해

객체나 배열이 다른 객체나 배열을 포함하는 경우, 좀 더 복잡한 패턴을 사용하면 중첩 배열이나 객체의 정보를 추출할 수 있다. 이를 *중첩 구조 분해*라고 한다.

아래 예시에서 대입 연산자 좌측의 패턴은 정보를 추출하려는 객체 `options`와 같은 구조를 가진다.

``` javascript
let options = {
    size : {
        width: 100,
        height: 200
    },
    items : ["Cake", "Donut"],
    extra : true
};

// 코드를 여러 줄에 걸쳐 작성하여 패턴을 정확하게 나타냄
let {
    size: {
        width,
        height
    }, 
    items: [item1, item2],
    title: "Menu" // 분해하려는 객체에 title 프로퍼티가 없으므로 기본값을 사용한다.
} = options;

alert(title); // Menu
alert(width); // 100
alert(height); // 200
alert(item1); // Cake
alert(item2); // Donut
```

`extra`를 제외한 `options` 객체의 모든 프로퍼티가 상응하는 변수에 할당되었다.

### 똑똑한 함수 매개변수

``` javascript
function showMenu(title = "Untitled", width = 200, height = 100, items = []) {
	//    ...
}
```

매개변수에 기본값이 설정되어 있어 굳이 인수를 넘겨주지 않아도 되는 경우에 아래 예시처럼 함수를 사용할 수 있다.

``` javascript
// 기본값을 사용해도 괜찮은 경우에는 매개변수에 undefined를 지정해주어야한다.
showMenu("My Menu", undefined, undefined, ["Item1", "Item2"]);
```

가독성이 떨어지고 코드가 지저분해지는 것을 볼 수 있다.

이런 문제를 구조 분해를 통해서 해결할 수 있다.

``` javascript
let options = {
    title: "My Menu",
    items: ["Item1", "Item2"]
};

// 전달받은 객체를 알아서 분해하여 변수에 적용
function showMenu (title = "Untitled", width = 200, height = 100, items = []) {
	// title, items = options에서 가져옴
    // width, height = 기본값을 지정
    alert(`${title}, ${width}, ${height}`); // My Menu, 200, 100
    alert(items); // Item1, Item2
}

showMenu(options);
```

중첩 객체와 콜론을 조합하여 좀 더 복잡하게 구조 분해를 할 수 있다.

``` javascript
let options = {
  title: "My menu",
  items: ["Item1", "Item2"]
};

function showMenu({
  title = "Untitled",
  width: w = 100,  // width는 w에,
  height: h = 200, // height는 h에,
  items: [item1, item2] // items의 첫 번째 요소는 item1에, 두 번째 요소는 item2에 할당함
}) {
  alert( `${title} ${w} ${h}` ); // My Menu 100 200
  alert( item1 ); // Item1
  alert( item2 ); // Item2
}

showMenu(options);
```
